# Quant-Agent

자율 주식 매매를 **최종 목표**로 하되, 처음부터 AI에게 매매를 맡기지 않는다.
먼저 **데이터 수집 → 분석 → 알림 → 레포팅**을 수행하는 순수 시스템을 단단히 만들고,
그 위에 **관찰·판단·제안** 에이전트를, 다시 그 위에 **자율 실행**을 단계적으로 얹는다.

> ⚠️ 이 프로젝트는 학습·실험용이다. 수익을 보장하지 않으며, 초기 단계에서 실거래는 하지 않는다.
> 자율 매매는 충분한 검증과 사람 승인 게이트를 거친 뒤에만, 소액으로 점진 도입한다.

## 핵심 설계 원칙

1. **단계적 신뢰 (Progressive Trust)** — 시스템이 신뢰를 "증명"한 만큼만 권한을 준다. 알림 → 제안 → 모의매매 → 소액 실매매.
2. **시스템 먼저, AI 나중 (System-first)** — Phase 0~1은 LLM이 전혀 들어가지 않는 결정론적 시스템. AI는 깨끗한 데이터 위에서만 동작한다.
3. **빠른 피드백 루프 (Fast Feedback)** — 보유기간이 짧은 스윙/단기(최대 2~3개월) 중심. 단위 시간당 완결된 매매 사이클을 많이 쌓아 전략·에이전트 개선 속도를 높인다. ("수익 최적"이 아니라 "학습 속도 최적"이 초기 목표다.)
4. **추상화로 미래 흡수 (Abstraction)** — 시장(KR/US), 데이터 소스, 알림 채널, 브로커를 전부 인터페이스 뒤에 둬서 교체·확장 비용을 낮춘다.
5. **불변·소형 모듈 (Immutable & Small)** — 데이터는 불변으로 다루고, 파일은 작고 응집도 높게.

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 언어 | Python | 퀀트·데이터·LLM 에이전트 생태계 표준 |
| 시세 (KR+US) | FinanceDataReader | 단일 API로 양 시장 커버 |
| 국내 공시 | DART OpenAPI | 무료, 공식 |
| 기술적 지표 | pandas-ta | 지표 재구현 불필요 |
| 저장 | DuckDB (초기) | 파일 1개로 시작, pandas와 자연스럽게 결합 |
| 스케줄 | APScheduler | 프로세스 내 스케줄링, 운영 단순 |
| 알림 | Discord (webhook) | 승인 버튼·멀티에이전트 페르소나·검색 히스토리에 유리 |
| 에이전트 (Phase 2+) | LangGraph (예정) | 멀티에이전트 그래프·상태 관리 |

## 대상 시장

- **국내(KR)** + **미국(US)** 동시. 시장 차이는 `collectors`/`universe` 추상화로 흡수.

## 단계 로드맵 요약

| Phase | 내용 | AI |
|-------|------|----|
| **0** | 데이터 수집·저장·스케줄링 | ❌ |
| **1** | 지표 분석 + 알림 + 정기 리포트 (규칙 기반) | ❌ |
| **2** | 산출물을 컨텍스트로 LLM 에이전트가 관찰·판단·제안 (paper) | ✅ 단일 |
| **3** | 멀티 에이전트 그룹 (펀더멘털/기술/리스크/PM 토론·합의) | ✅ 그룹 |
| **4** | 소액 위임 + 사람 승인 게이트 → 점진 자율화 | ✅ 실행 |

상세는 [docs/roadmap.md](docs/roadmap.md) 참고.
