# 유니버스 전략

## 왜 단기/스윙인가 — 솔직한 전제

**단기/스윙이 장기보다 "수익내기 쉬워서"가 아니다.** 오히려 노이즈·거래비용·세금·HFT 경쟁
때문에 단기 수익은 *더 어렵다.* 그럼에도 초기에 단기/스윙(최대 2~3개월)을 선택하는 이유는
**개발 피드백 속도** 때문이다.

- 보유기간이 짧으면 **단위 시간당 완결되는 매매 사이클**(= 라벨링된 결과)이 훨씬 많이 쌓인다.
- 전략·에이전트의 가설 → 검증 → 개선 루프가 빨라진다.
- 백테스트 통계 유의성을 빨리 확보한다.

즉 초기 목표는 **"수익 최적"이 아니라 "학습 속도 최적"**이다. 이 프레임 위에서 유니버스를 설계한다.

## 보유기간 정의

| 분류 | 보유기간 | 주된 신호 |
|------|----------|-----------|
| 단기 | 며칠 ~ 2주 | 기술적 모멘텀, 뉴스 촉매 |
| 스윙 | 2주 ~ 2개월 | 추세 + 실적/이벤트 |
| (상한) | 최대 2~3개월 | 이 이상은 초기 범위 밖 |

## 스윙에 적합한 티커의 조건

좋은 유니버스는 종목 리스트가 아니라 **선정 기준**이다. 기준을 config로 두면, 나중에 LLM이
근거를 갖고 "유니버스에 X 추가/제거"를 제안할 수 있다.

1. **유동성** — 깨끗한 진입/청산. 충분한 일평균 거래대금. 잡주·동전주 제외.
2. **변동성 스윗스팟** — 일간 ATR 약 2~5%. 너무 낮으면 기회 없음, 너무 높으면 노이즈.
3. **촉매 빈도** — 분기 실적 + 활발한 뉴스 흐름. 움직일 이유가 정기적으로 있는 종목.
4. **기술적 정직성** — 지지/저항·추세를 존중. 휩쏘(whipsaw) 심한 종목 제외.
5. **상관관계 분산** — 한 섹터에 몰리지 않게. 신호 다양성 확보.

## 티어 구조

하드코딩 리스트 대신 **2-티어 + 조건 기반**으로 구성한다.

### Tier 1 — 코어 (~25종목, 수동 큐레이션, 항상 추적)
유동성·변동성이 검증된 이름. 파이프라인이 매일 안정적으로 도는 기준선.

### Tier 2 — 동적 스캔 (주기적 갱신, 로테이션)
거래대금 상위 / 등락률 상위 풀에서 위 5개 조건을 통과한 종목을 자동 편입·이탈.
`universe/screener.py`가 담당. Tier 1을 보조하며 새 기회를 발굴.

## 스크리너 동작 (Tier 2, 목표)

```
screener.scan(market):
    후보 = 거래대금_상위(market, N) ∪ 등락률_상위(market, N)
    for 종목 in 후보:
        지표 = analysis.compute(종목)        # ATR, 거래대금, 추세 점수
        if 유동성_충족 and ATR in [2%, 5%] and not 휩쏘:
            편입_후보에 추가
    return 상위_M개 (상관관계 분산 적용)
```

선정/탈락 사유를 구조화 로그로 남긴다 → Phase 2에서 LLM이 "왜 이 종목이 들어왔나"를 설명·재평가하는 컨텍스트가 된다.
